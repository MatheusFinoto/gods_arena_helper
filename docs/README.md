# Docs

Esta pasta centraliza a documentacao funcional e tecnica do projeto.

Objetivo:

- registrar fluxos importantes do sistema;
- manter padroes de implementacao simples e visiveis;
- especificar novas features antes do desenvolvimento;
- criar uma base de consulta para futuras evolucoes.

## Estrutura inicial

- `flows-and-standards.md`: resumo dos fluxos atuais e combinados iniciais do projeto.
- `features/settings.md`: especificacao inicial da aba `Settings`.

## Regra de uso desta pasta

Sempre que uma nova feature for iniciada, devemos primeiro registrar:

1. objetivo da feature;
2. comportamento esperado;
3. regras de negocio;
4. impacto em frontend e backend;
5. riscos ou dependencias.

Assim, a pasta `docs` vira a referencia principal para evolucao do projeto.
